Enroll now →</a></p></div>\t\t\t\t\t\t\t\t<div style=\"overflow: hidden;width: 0px;height: 0px\"><img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/lady.jpg?x72090\"><br>\n<img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/lady-map.jpg?x72090\"><br>\n<img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/ball-map.jpg?x72090\"><br>\n<img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/ball.jpg?x72090\"><br>\n<img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/mount.jpg?x72090\"><br>\n<img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/mount-map.jpg?x72090\"><br>\n<img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/canyon-map.jpg?x72090\"><br>\n<img decoding=\"async\" style=\"width: 0;height: 0\" src=\"https://tympanus.net/Development/Fake3DEffect/img/canyon.jpg?x72090\"></div>\n<p>WebGL is becoming quite popular these days as it allows us to create unique interactive graphics for the web. You might have seen the recent <a href=\"https://tympanus.net/codrops/2019/02/06/text-distortion-effects-using-blotter-js/\">text distortion effects using Blotter.js</a> or the <a href=\"https://tympanus.net/codrops/2019/01/08/animated-mesh-lines/\">animated WebGL lines created with the THREE.MeshLine library</a>. Today you’ll see how to quickly create an interactive “fake” 3D effect for images with plain WebGL.</p>\n<p>If you use Facebook, you might have seen the update of <a href=\"https://facebook360.fb.com/2018/10/11/3d-photos-now-rolling-out-on-facebook-and-in-vr/\">3D photos for the news feed and VR</a>. With special phone cameras that capture the distance between the subject in the foreground and the background, 3D photos bring scenes to life with depth and movement. We can recreate this kind of effect with any photo, some image editing and a little bit of coding.</p>\n<p>Usually, these kind of effects would rely on either Three.js or Pixi.js, the powerful libraries that come with many useful features and simplifications when coding. Today we won’t use any libraries but go with the native WebGL API.</p>\n<p>So let’s dig in.</p>\n<h2>Getting started</h2>\n<p>So, for this effect we’ll go with the native WebGL API. A great place to help you get started with WebGL is <a href=\"https://webglfundamentals.org\">webglfundamentals.org</a>. WebGL is usually being berated for its verboseness. And there is a reason for that. The foundation of all fullcreen shader effects (even if they are 2D) is some sort of plane or mesh, or so called quad, which is stretched over the whole screen. So, speaking of being verbose, while we would simply write <code class=\"\" data-line=\"\">THREE.PlaneGeometry(1,1)</code> in three.js which creates the 1×1 plane, here is what we need in plain WebGL:</p>\n<pre class=\"brush:js\"><code class=\"\" data-line=\"\">let vertices = new Float32Array([\n\t  -1, -1,\n\t  1, -1,\n\t  -1, 1,\n\t  1, 1,\n\t])\n\tlet buffer = gl.createBuffer();\n\tgl.bindBuffer( gl.ARRAY_BUFFER, buffer );\n\tgl.bufferData( gl.ARRAY_BUFFER, vertices, gl.STATIC_DRAW );</code></pre>\n<p>Now that we have our plane, we can apply vertex and fragment shaders to it.</p>\n<h2>Preparing the image</h2>\n<p>For our effect to work, we need to create a depth map of the image. The main principle for building a depth map is that we’ve got to separate some parts of the image depending on their Z position, i.e. being far or close, hence isolate the foreground from the background.</p>\n<p>For that, we can open the image in Photoshop and paint gray areas over the original photo in the following way:</p>\n<p><img decoding=\"async\" class=\"alignnone size-full wp-image-38429\" src=\"https://codrops-1f606.kxcdn.com/codrops/wp-content/uploads/2019/02/fake3d_01.gif?x72090\" alt=\"fake3d_01\" width=\"800\" height=\"534\"></p>\n<p>This image shows some mountains where you can see that the closer the objects are to the camera, the brighter the area is painted in the depth map. Let’s see in the next section why this kind of shading makes sense.</p>\n<div class=\"box-info\"><p><em>Tiny break: </em>Want to get website inspiration? <strong><a href=\"https://tympanus.net/codrops/news/\" target=\"_blank\">Check out our Webzibition</a> and see whats trending.</strong></p></div>\n<h2>Shaders</h2>\n<p>The rendering logic is mostly happening in shaders. As described in the MDN web docs:</p>\n<blockquote><p>A shader is a program, written using the OpenGL ES Shading Language (GLSL), that takes information about the vertices that make up a shape and generates the data needed to render the pixels onto the screen: namely, the positions of the pixels and their colors. There are two shader functions run when drawing WebGL content: the vertex shader and the fragment shader.</p></blockquote>\n<p>A great resource to learn more about shaders is <a href=\"https://thebookofshaders.com\">The Book Of Shaders</a>.</p>\n<p>The vertex shader will not do much; it just shows the vertices:</p>\n<pre class=\"brush:glsl\"><code class=\"\" data-line=\"\">attribute vec2 position;\n    void main() {\n    gl_Position = vec4( position, 0, 1 );\n}</code></pre>\n<p>The most interesting part will happen in a fragment shader. Let’s load the two images there:</p>\n<pre class=\"brush:glsl\"><code class=\"\" data-line=\"\">void main(){\n    vec4 depth = texture2D(depthImage, uv);\n    gl_FragColor = texture2D(originalImage, uv); // just showing original photo\n}</code></pre>\n<p>Remember, the depth map image is black and white. For shaders, color is just a number: 1 is white and 0 is pitch black. The <code class=\"\" data-line=\"\">uv</code> variable is a two dimensional map storing information on which pixel to show. With these two things we can use the depth information to move the pixels of the original photo a little bit.</p>\n<p>Let’s start with a mouse movement:</p>\n<pre class=\"brush:glsl\"><code class=\"\" data-line=\"\">vec4 depth = texture2D(depthImage, uv);\ngl_FragColor = texture2D(originalImage, uv + mouse);</code></pre>\n<p>Here is how it looks like:</p>\n<p><img decoding=\"async\" class=\"alignnone size-full wp-image-38428\" src=\"https://codrops-1f606.kxcdn.com/codrops/wp-content/uploads/2019/02/fake3d_02.gif?x72090\" alt=\"fake3d_02\" width=\"726\" height=\"454\"></p>\n<p>Now let’s add the depth:</p>\n<pre class=\"brush:glsl\"><code class=\"\" data-line=\"\">vec4 depth = texture2D(depthImage, uv);\ngl_FragColor = texture2D(originalImage, uv + mouse*depth.r);</code></pre>\n<p>And here we are:</p>\n<p><img loading=\"lazy\" decoding=\"async\" class=\"alignnone size-full wp-image-38427\" src=\"https://codrops-1f606.kxcdn.com/codrops/wp-content/uploads/2019/02/fake3d_03.gif?x72090\" alt=\"fake3d_03\" width=\"726\" height=\"454\"></p>\n<p>Because the texture is black and white, we can just take the red channel (<code class=\"\" data-line=\"\">depth.r</code>), and multiply it to the mouse position value on the screen. That means, the brighter the pixel is, the more it will move with the mouse. On the other hand, dark pixels will just stay in place. It’s so simple, yet, it results in such a nice 3D illusion of an image.</p>\n<p>Of course, shaders are capable of doing all kinds of other crazy things, but I hope you like this small experiment of “faking” a 3D movement. Let me know what you think about it, and I hope to see your creations with this!</p>\n<p><!-- Can be added in the end of the article --></p>\n<h2>References and Credits</h2>\n<ul>\n<li><a
