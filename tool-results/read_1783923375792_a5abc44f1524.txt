We MUST replicate this or the effect looks cheap.
    20→    20→- **Critical insight #2**: Limit camera/mouse motion to a small range (~±10° rotation or a few tens of px translation). Google's Cinematic Photos team confirmed this is the key to hiding artifacts without expensive per-frame inpainting.

    51→    51→### 2. Layer Separation from Depth Maps
    52→    52→
    53→    53→Given a continuous grayscale depth map, splitting into N discrete layers:
    54→    54→
    55→    55→**a) Equal-width thresholding (simplest)**: normalize depth to [0,1], slice into N bands. Easy but tends to over-segment smooth gradients.
    56→    56→
    57→    57→**b) K-means / quantile clustering** (recommended for 3–5 layers): cluster depth pixels into K bins using `sklearn.cluster.KMeans` or 1-D k-means on the depth histogram. Adapts to actual scene depth distribution. Reference: "Object-Background Region Separation in Depth Image using k-Means Clustering" (KISM 2021).
    58→    58→
    59→    59→**c) Bilateral filtering first** (used by Shih et al. 3D-Photo-Inpainting): edge-preserving smoothing of the depth map before clustering → cleaner layer boundaries that align with image edges.
    60→    60→
    61→    61→**d) Watershed / connected-component labeling**: produce spatially coherent regions (not just depth bands) → useful for "subject vs background" rather than "near vs far".
    62→    62→
    63→    63→**e) Adaptive layering (Layered Depth Image, LDI)**: the gold standard from Shih et al. — splits along depth *discontinuities* with explicit pixel connectivity, producing a graph of front-to-back layers each with its own inpainted color+depth. This is what 3D Photo Inpainting / Immersity use.
    64→    64→
    65→    65→For our app, **b + c (bilateral-filter then 1-D k-means with K=3–5)** is the practical sweet spot for "foreground / midground / background / sky" decomposition. Save the cluster centroids as depth thresholds and emit per-layer masks as PNGs.
    66→    66→
    67→    67→### 3. Segmentation-Based Separation (SAM / SAM2)
    68→    68→
    69→    69→**Meta SAM2** (`facebookresearch/sam2`, Apache 2.0) is the SOTA. Two modes:
    70→    70→- **Promptable**: click/box/mask → precise mask. Great for "user clicks subject".
    71→    71→- **Automatic mask generator**: produces all plausible masks. Then we filter by area / centrality to pick the main subject.
    72→    72→
    73→    73→**Sky detection**: dedicated models exist (SkyFinder dataset, "Segmenting Sky Pixels in Images"), but a simpler heuristic often suffices — combine (a) top-of-image region, (b) blue-ish / bright color, (c) **far depth** (depth value in the top decile). Can also use a semantic seg model like **ADE20K-trained OneFormer / Mask2Former** which has a "sky" class out of the box.
    74→    74→
    75→    75→**Ground detection**: similar — bottom-of-image + dark/uniform color + near depth. Or ADE20K class "floor / grass / road / earth".
    76→    76→
    77→    77→**Subject detection**: SAM2 auto-mask → pick the largest non-sky/non-ground mask whose centroid is in the central 60% of the frame. Optionally refine edges with **Matting-Anything** or **MODNet** for hair-level fidelity (important for portraits).
    78→    78→
    79→    79→**Recommended pipeline**:
    80→    80→1. Run ADE20K semantic seg (1 forward pass) → labels every pixel as sky / ground / person / building / etc.
    81→    81→2. Run SAM2 auto-mask → instance-level subject candidates.
    82→    82→3. Cross-reference: subject = SAM2 mask overlapping "person" or central non-background ADE20K labels.
    83→    83→
    84→    84→For a Next.js app, run server-side via the `transformers.js` SAM2 port or a Python microservice. **transformers.js** can run SAM2 in Node.js directly — no Python needed.
    85→    85→
    86→    86→### 4. Background Removal + Inpainting
    87→    87→
    88→    88→**Background removal**:
    89→    89→- **Rembg** (Python, MIT) — wraps U2Net / BRIA RMBG / IS-Net. CLI + library. Production-grade, ~1–2s per image on CPU. Best free option.
    90→    90→- **@imgly/background-removal** (JS, runs in browser or Node via WASM) — drop-in for Next.js, no Python.
    91→    91→
    92→    92→**Inpainting the occluded background** (critical for parallax quality):
    93→    93→- **LaMa** (Resolution-robust Large Mask Inpainting with Fourier Convolutions, Samsung Research) — SOTA for large mask inpainting, generalizes to 2K res despite 256×256 training. Fast (~1s/image on GPU). Apache 2.0.
    94→    94→- **IOPaint** (formerly Lama-Cleaner, MIT) — self-hostable web app + Python lib + HTTP API that wraps LaMa, Stable Diffusion inpainting, MAT, ZITS, FcF, Manga, PaintByExample, and several matting models. **This is the swiss-army knife.** Docker container → Next.js calls `POST http://iopaint:8000/inpaint` with image + mask.
    95→    95→- **Stable Diffusion Inpainting** — highest quality for complex scenes, but slow (5–15s on GPU) and may "hallucinate" extra content.
    96→    96→- **3D Photo Inpainting** (Shih et al., CVPR 2020, `vt-vl-lab/3d-photo-inpainting`) — the canonical method. Takes RGB+depth, outputs a Layered Depth Image (LDI) with color+depth for occluded regions. Heavy (~10 min on A100 per image via Replicate). Available as `pollinations/3d-photo-inpainting` on Replicate. **Best quality**, expensive.
    97→    97→- **SLIDE** (ICCV 2021) — soft layering variant; handles hair/foliage better. No official code; community reimplementations exist.
    98→    98→
    99→    99→**Practical recommendation for our app**:
   100→   100→1. Default: Rembg (or `@imgly/background-removal`) for subject alpha matte.
   101→   101→2. **Dilate the subject mask by ~20–40 px** (this is what Facebook 3D Photos / Tiefling's `expandDepthmapRadius` do — exposes a band of background that can be revealed when subject shifts).
   102→   102→3. Run **LaMa via IOPaint** on the dilated mask → reconstructs background where the subject used to be. Store this as `background_inpainted.png`.
   103→   103→4. For premium mode: route through `pollinations/3d-photo-inpainting` on Replicate for the full LDI treatment (cached, async).

   149→   149→### 7. The "LSD Alive" Effect — Subtle Animation Toolkit
   150→   150→
   151→   151→The "alive" feeling is the *combination* of several subtle, low-amplitude animations. Each alone is barely noticeable; together they create the magic. Key techniques (ranked by impact):
   152→   152→
   153→   153→1. **Mouse / gyro parallax** (the foundation). Map `mouseX/Y` (or `deviceorientation`) to camera/layer offset, multiplied by per-layer depth. Smooth with lerp (damping ~0.05–0.1).
   154→   154→2. **Breathing / pulsing** — slow (4–8s period) `scale: 1.0 ↔ 1.015`. Apply to whole image and slightly stronger to foreground.
   155→   155→3. **Sway** — gentle `rotate: ±0.3°` oscillation on a 6–10s sine. Subtle but very "alive".
   156→   156→4. **Liquid / wavy distortion** — `<feTurbulence>` SVG filter animated via `seed` or `baseFrequency`, applied with low `scale` (1–3px displacement). Gives the "water surface" shimmer. Optional per-region.
   157→   157→5. **Floating particles** — CSS / canvas particles (dust motes, light specks) drifting upward or with mouse. ~10–30 particles, low opacity.
   158→   158→6. **Light shimmer** — slow gradient overlay sweeping across (radial highlight following mouse, or auto-cycling). ~2–5% opacity.
   159→   159→7. **Chromatic aberration / RGB split** — shift R/B channels 1–3px based on mouse distance from center. Increases at edges (radial).
   160→   160→8. **Depth-of-field shift** — blur the far layer (or near layer) alternately on a 8–12s cycle. `filter: blur(0–4px)`.
   161→   161→9. **Ken Burns** — slow zoom (1.0 → 1.05 over 20–30s) + sub-pixel pan. Good as a base when there's no mouse input.
   162→   162→10. **Vignette pulse** — very subtle darkening at edges pulsing with breathing.
   163→   163→
   164→   164→Amplitude guidelines: any single effect should be barely perceptible when stared at. Combined, they shouldn't draw attention to themselves. The user should just feel "the image is breathing".

   166→   166→### 8. Code Examples
   167→   167→
   168→   168→**a) Codrops / canonical WebGL fragment shader** (the foundation of LeiaPix/DepthFlow/Tiefling):
   169→   169→```glsl
   170→   170→// Fragment shader: original image + depth map → parallax with mouse
   171→   171→uniform sampler2D originalImage;
   172→   172→uniform sampler2D depthImage;
   173→   173→uniform vec2 mouse;  // normalized -1..1, damped
   174→   174→uniform float depthStrength;  // ~0.02 for subtle
   175→   175→varying vec2 vUv;
   176→   176→
   177→   177→void main() {
   178→   178→    vec4 depth = texture2D(depthImage, vUv);
   179→   179→    // White = close → moves more with mouse; black = far → stays put
   180→   180→    vec2 offset = mouse * depth.r * depthStrength;
   181→   181→    gl_FragColor = texture2D(originalImage, vUv + offset);
   182→   182→}
   183→   183→```
   184→   184→That's the whole effect. ~6 lines of GLSL. From the Codrops tutorial by Yuri Artiukh.
   185→   185→
   186→   186→**b) React Three Fiber implementation** (R3F + drei):
   187→   187→```tsx
   188→   188→import { Canvas, useFrame, useLoader } from '@react-three/fiber'
   189→   189→import { TextureLoader } from 'three'
   190→   190→import { useRef, useMemo } from 'react'
   191→   191→import * as THREE from 'three'
   192→   192→
   193→   193→const vertexShader = `
   194→   194→  varying vec2 vUv;
   195→   195→  void main() { vUv = uv; gl_Position = vec4(position, 1.0); }
   196→   196→`
   197→   197→const fragmentShader = `
   198→   198→  uniform sampler2D uImage; uniform sampler2D uDepth;
   199→   199→  uniform vec2 uMouse; uniform float uTime;
   200→   200→  varying vec2 vUv;
   201→   201→  void main() {
   202→   202→    vec4 d = texture2D(uDepth, vUv);
   203→   203→    // subtle breathing + mouse parallax
   204→   204→    float breath = sin(uTime * 0.8) * 0.003 * d.r;
   205→   205→    vec2 off = uMouse * d.r * 0.03 + vec2(breath, breath * 0.5);
   206→   206→    gl_FragColor = texture2D(uImage, vUv + off);
   207→   207→  }
   208→   208→`
   209→   209→
   210→   210→function AlivePlane({ img, depth }) {
   211→   211→  const mat = useRef<THREE.ShaderMaterial>(null!)
   212→   212→  const tex = useLoader(TextureLoader, img)
   213→   213→  const depthTex = useLoader(TextureLoader, depth)
   214→   214→  const mouse = useMemo(() => new THREE.Vector2(0, 0), [])
   215→   215→  useFrame((state, delta) => {
   216→   216→    // smooth mouse follow
   217→   217→    mouse.x += (state.pointer.x - mouse.x) * 0.05
   218→   218→    mouse.y += (state.pointer.y - mouse.y) * 0.05
   219→   219→    mat.current.uniforms.uMouse.value = mouse
   220→   220→    mat.current.uniforms.uTime.value += delta
   221→   221→  })
   222→   222→  return (
   223→   223→    <mesh>
   224→   224→      <planeGeometry args={[2, 2]} />
   225→   225→      <shaderMaterial ref={mat} vertexShader={vertexShader} fragmentShader={fragmentShader}
   226→   226→        uniforms={{
   227→   227→          uImage:  { value: tex },
   228→   228→          uDepth:  { value: depthTex },
   229→   229→          uMouse:  { value: mouse },
   230→   230→          uTime:   { value: 0 },
   231→   231→        }} />
   232→   232→    </mesh>
   233→   233→  )
   234→   234→}
   235→   235→```

   266→   266→```js
   267→   267→const layers = document.querySelectorAll('.layer')
   268→   268→const state = { mx: 0, my: 0, tx: 0, ty: 0 }
   269→   269→window.addEventListener('pointermove', e => {
   270→   270→  state.mx = (e.clientX / innerWidth - 0.5)

   272→   272→})
   273→   273→gsap.ticker.add(() => {
   274→   274→  state.tx += (state.mx - state.tx) * 0.06
   275→   275→  state.ty += (state.my - state.ty) * 0.06
   276→   276→  layers.forEach(l => {
   277→   277→    const d = parseFloat(l.dataset.depth)
   278→   278→    gsap.set(l, { x: state.tx * 40 * d, y: state.ty * 40 * d,
   279→   279→                  rotation: state.tx * 1.5 * d })
   280→   280→  })
   281→   281→})
   282→   282→// breathing pulse on whole scene
   283→   283→gsap.to('.scene', { scale: 1.015, duration: 4, yoyo: true, repeat: -1, ease: 'sine.inOut' })

   286→   286→**e) SVG turbulence for liquid shimmer** (overlay on the image):
   287→   287→```html

   312→   312→### 9. Depth-Map-to-Displacement Technique — Deep Dive (the LeiaPix/Immersity method)
   313→   313→
   314→   314→This is the single most important technique for our project. Two rendering variants:
   315→   315→
   316→   316→**Variant A — Fragment-shader UV offset** (Codrops, Depthy, Pixi DisplacementFilter):
   317→   317→- 1 fullscreen quad, fragment shader samples depth, offsets UV.
   318→   318→- ✅ Trivial to implement, ~6 lines GLSL, runs at 60fps on any GPU.
   319→   319→- ❌ Produces **"rubber band" / stretch artifacts** at depth discontinuities: when a foreground pixel shifts, it leaves a hole filled by the next pixel's color → smeared edges.
   320→   320→
   321→   321→**Variant B — Vertex-displaced mesh** (Tiefling, three.js forum examples):
   322→   322→- Subdivided plane geometry; each vertex's Z = depth value.
   323→   323→- ✅ True parallax geometry, handles occlusion correctly via depth buffer.
   324→   324→- ❌ Mesh resolution tradeoff: vertex per pixel = too many verts (2M for 1080p). Need adaptive meshing.
   325→   325→
   326→   326→**Variant C — Layered / multi-plane** (Facebook 3D Photo, Immersity, 3D-Photo-Inpainting):
   327→   327→- N depth planes, each with its own (potentially inpainted) color texture.
   328→   328→- ✅ Best quality: occlusion solved by inpainting behind each foreground layer.
   329→   329→- ❌ Requires per-layer background inpainting; more complex asset pipeline.
   330→   330→
   331→   331→**The key quality unlocks (do ALL of these)**:
   332→   332→1. **Dilate the depth map at object edges** before using as displacement (Tiefling's `expandDepthmapRadius`). This pre-extends the background so when foreground shifts, there's real background to show.
   333→   333→2. **Inpaint the background** behind the main subject (LaMa). Ship `background_inpainted.png` as a separate texture sampled where `depth < subject_depth`.
   334→   334→3. **Limit motion range** (Google Cinematic Photos insight): clamp mouse offset to ±10° or ±30–50 px. Beyond that, artifacts become visible.
   335→   335→4. **Use easing / damping** on mouse so motion never exceeds comfortable range.
   336→   336→5. **Apply bilateral filtering** to the depth map to remove noisy speckles before slicing.
   337→   337→6. For best results, render with **variant B (vertex mesh) using the inpainted background as a second layer behind** — gets ~90% of Immersity quality at 10% of the compute.
   338→   338→
   339→   339→**Concrete shader with occlusion-aware background fill** (Variant C, simplified):
   340→   340→```glsl
   341→   341→uniform sampler2D uForeground;   // original image
   342→   342→uniform sampler2D uBackground;   // LaMa-inpainted background (subject removed)
   343→   343→uniform sampler2D uDepth;
   344→   344→uniform sampler2D uSubjectMask;  // 1=subject, 0=background
   345→   345→uniform vec2 uMouse;
   346→   346→uniform float uTime;
   347→   347→varying vec2 vUv;
   348→   348→
   349→   349→void main() {
   350→   350→  vec4 d = texture2D(uDepth, vUv);
   351→   351→  float mask = texture2D(uSubjectMask, vUv).r;
   352→   352→  float breath = sin(uTime * 0.6) * 0.002;
   353→   353→
   354→   354→  // Foreground shifts more (subject), background less
   355→   355→  vec2 bgOff = uMouse * (1.0 - d.r) * 0.015 + vec2(breath);
   356→   356→  vec2 fgOff = uMouse * d.r * 0.05 + vec2(breath * 2.0);
   357→   357→
   358→   358→  vec3 bg = texture2D(uBackground, vUv + bgOff).rgb;
   359→   359→  vec3 fg = texture2D(uForeground, vUv + fgOff).rgb;
   360→   360→
   361→   361→  // Use the displaced mask to decide which layer shows
   362→   362→  float displacedMask = texture2D(uSubjectMask, vUv + fgOff).r;
   363→   363→  vec3 color = mix(bg, fg, displacedMask);
   364→   364→
   365→   365→  gl_FragColor = vec4(color, 1.0);
   366→   366→}
   367→   367→```

   372→   372→
   373→   373→### 🥇 Rank 1 — Hybrid: Depth Anything V2 + LaMa + Three.js shader (RECOMMENDED)
   374→   374→**Quality: 9/10 | Feasibility: 8/10 | Effort: Medium | Cost: Low**
   375→   375→
   376→   376→Pipeline:
   377→   377→1. Upload image → Next.js Route Handler.
   378→   378→2. **Server-side (Node, via `@xenova/transformers` or Python microservice)**:
   379→   379→   - Run **Depth Anything V2 Small/Base** → depth map PNG.
   380→   380→   - Run **SAM2 auto-mask** → main subject mask.
   381→   381→   - Run **LaMa (via IOPaint Docker)** on (image, dilated subject mask) → inpainted background.
   382→   382→   - Optionally: bilateral filter the depth, dilate subject mask by ~25 px.
   383→   383→3. Store 4 assets: `original.png`, `depth.png`, `subject_mask.png`, `background_inpainted.png`. Persist URLs in Prisma.
   384→   384→4. **Client-side (R3F shader)**: Variant C shader above + GSAP breathing + SVG turbulence overlay + chromatic aberration. Mouse-driven parallax clamped to ±0.05 UV.
   385→   385→
   386→   386→**Why #1**: Best quality-per-effort. Uses SOTA depth + SOTA inpainting without paying for 3D-Photo-Inpainting's 10-min/A100 cost. The `transformers.js` route means **no Python required** if we use `@xenova/transformers` + `@imgly/background-removal` + IOPaint-as-a-service. Total compute: ~2–4s/image on a CPU-only Vercel-style box, <1s on GPU.
   387→   387→
   388→   388→### 🥈 Rank 2 — Pure browser-side (Tiefling-style)
   389→   389→**Quality: 7/10 | Feasibility: 9/10 | Effort: Low | Cost: Free (no server GPU)**
   390→   390→
   391→   391→Pipeline:
   392→   392→1. Upload image → stays in browser.
   393→   393→2. `onnxruntime-web` (WebGPU) runs **Depth Anything V2 Small q4f16** (~18MB) → depth map. ~3–5s on M1 Mac, ~20s on mid hardware.
   394→   394→3. `@imgly/background-removal` (WASM) → subject mask + alpha. ~2–3s.
   395→   395→4. Skip LaMa (no good browser inpainting yet); instead use **edge dilation** on the depth + clamp mouse range hard.
   396→   396→5. R3F shader (Variant A) — accept minor stretch artifacts in exchange for zero infra.
   397→   397→
   398→   398→**Why #2**: Zero server compute cost, fully private (image never leaves browser). Great for a free tier / demo. Quality ceiling lower due to no real inpainting. Tiefling's open-source MIT code is a direct reference implementation.
   399→   399→
   400→   400→### 🥉 Rank 3 — Cloud API (Immersity AI or Replicate)
   401→   401→**Quality: 10/10 | Feasibility: 9/10 | Effort: Very Low | Cost: $$**
   402→   402→
   403→   403→Pipeline:
   404→   404→1. Upload → POST to Immersity AI API (or `pollinations/3d-photo-inpainting` on Replicate).
   405→   405→2. Receive: depth map + layered 3D data, OR pre-rendered video.
   406→   406→3. If interactive: render with their viewer / our shader using their layered output. If video: just `<video autoplay loop muted>`.
   407→   407→
   408→   408→**Why #3**: Best quality bar none, least engineering. But: per-image cost ($0.05–0.50), privacy concerns (user images uploaded to third party), rate limits, and vendor lock-in. Good as a **premium upsell** ("Pro" mode) layered on top of Rank 1.
   409→   409→
   410→   410→### Rank 4 — Stable Diffusion Inpainting for background
   411→   411→**Quality: 9/10 | Feasibility: 6/10 | Effort: Medium-High | Cost: Medium**
   412→   412→
   413→   413→Like Rank 1 but replace LaMa with **SD Inpainting** (or SDXL inpaint). Higher quality reconstruction (especially for complex backgrounds like foliage, patterns). But 5–15s/image on GPU and risk of hallucinated content. Use only for premium tier.
   414→   414→
   415→   415→### Rank 5 — Hard 3-layer CSS-only parallax (no depth model)
   416→   416→**Quality: 5/10 | Feasibility: 10/10 | Effort: Very Low | Cost: Free**
   417→   417→
   418→   418→Skip depth model entirely. User (or simple auto-segmentation) provides 3 layers as separate PNGs: sky / midground / subject. Animate with GSAP Variant (d). Cheapest, fastest, but quality limited and requires either manual prep or Rembg + manual layering.
